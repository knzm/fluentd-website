設定ファイル¶

設定ファイルは、ユーザーに(1)input/outputプラグインの選択、(2)プラグインパラメータの指定によるFluentdの入出力の振る舞いのコントロールを提供します。このファイルはFluentdを正常に使用するために必要不可欠です。

設定ファイルは $install_prefix/etc/fluent/fluent.conf に置かれます。ファイルが存在しない場合、ユーザーは次のコマンドで作成する必要があります:

$ sudo fluentd --setup /etc/fluent
$ edit /etc/fluent/fluent.conf

設定ファイルには以下が含まれている必要があります:

  1. <source> ディレクティブ : 入力元の決定
  2. <match> ディレクティブ : 出力先の決定

このドキュメントでは、設定ファイルの書式について説明します。

  • <source> ディレクティブ
  • <match> ディレクティブ
    • マッチパターン
  • インクルード

<source> ディレクティブ¶

Fluentdの入力元は <source> ディレクティブを使用して使いたいインプットプラグインを選択し設定することにより有効になります。Fluentdは標準的なインプットプラグインとして http と forward を含んでいます。

各 <source> ディレクティブは type パラメータを含んでいる必要があります。 type パラメータは選択した入力プラグインを指定します。

例:

# Receive events from 24224/tcp
# This is used by log forwarding and fluent-cat command
<source>
  type forward
  port 24224
</source>

# http://this.host:9880/myapp.access?json={"event":"data"}
<source>
  type http
  port 9880
</source>

ユーザーは独自のプラグインを作成することによってFluentdの入力元を元々提供されているものから拡張することができます。Fluentdの入力元に関する詳細情報については、 入力プラグイン を参照してください。

ソースはFluentdのルーティグエンジンにイベントを送信します。イベントは、タグ(tag), 日時(time), レコード(record)という3つの要素で構成されます。タグは’.’で区切られた文字列(例: myapp.access)で、Fluentd内部のルーティグエンジンによって使われます。日時はイベントが発生したUNIX timeです。レコードはJSONオブジェクトです。上の例では、forwardプラグインは次のイベントを送信します:

tag: myapp.access
time: (current time)
record: {"event":"data"}

<match> ディレクティブ¶

Fluentdの出力先は <match> ディレクティブを使用して使いたいアウトプットプラグインを選択し設定することにより有効になります。Fluentdは標準的なアウトプットプラグインとして file と forward を含んでいます。

各 <match> ディレクティブは type パラメータとマッチパターンを含んでいる必要があります。マッチパターンはイベントのフィルターとして使用されます。イベントのタグ(tag)がパターンとマッチした場合のみ出力先に送られます。 type パラメータは選択した出力プラグインを指定します。

例えば、ユーザーは myapp.accesslog.** パターンに一致するすべてを、指定したディレクトリ内のファイルへ送ることができます。

例:

# Match events tagged with "myapp.access" and
# store them to /var/log/fluent/access.%Y-%m-%d
<match myapp.access>
  type file
  path /var/log/fluent/access
</match>

<match myapp.log.**>
  type file
  format /var/log/fluent/myapp_hourly
  time_slice_format %Y%m%d%H
</match>

ユーザーは独自のプラグインを作成することによってFluentdの入力元を元々提供されているものから拡張することができます。Fluentdの出力先に関する詳細情報については、 :ref:`output_plugin ` を参照してください。

マッチパターン¶

次のマッチパターンがユーザーに提供されています:

  • * は１つのtag要素とマッチします。
    • 例えば、 a.* は a.b と一致しますが、 a や a.b.c とは一致しません。
  • ** はゼロ個以上のtag要素と一致します。
    • 例えば、 a.** は a, a.b, a.b.c と一致します。
  • {X,Y,Z} は X, Y, Z にマッチします、なお X, Y, Z はマッチパターンです。
    • 例えば、 {a,b} というパターンは a と b にはマッチしますが、 c にはマッチしません。
    • これは * や ** のパターンと組み合わせて使うことができます。例： a.{b,c}.* や a.{b,c.**}

インクルード¶

分割された設定ファイルのディレクティブは include ディレクティブを使用して読み込むことができます。

# Include config files in ./config.d directory
include config.d/*.conf

include ディレクティブは、通常のファイルpath, globパターン, HTTP URLをサポートします。

# absolute path
include /path/to/config.conf

# if using a relative path, the directive will use
# the dirname of this file to expand the path
include extra.conf

# glob match pattern
include config.d/*.conf

# http
include http://example.com/fluent.conf
